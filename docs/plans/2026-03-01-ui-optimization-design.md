# UI 优化设计方案

**Date**: 2026-03-01
**Status**: Approved

## 1. 项目概述

对 OA Proxy 桌面应用进行全面 UI 优化和功能补全，包括视觉风格升级、组件标准化、服务状态控制等功能。

## 2. 现状分析

### 当前问题
- 服务运行状态未在 UI 中展示
- 缺少启动/停止服务的交互按钮
- 规则表单保存按钮样式不明显
- 分组删除功能位置不当（隐藏在 group header 中）
- 输入框、按钮、导航等组件样式不统一
- 整体视觉风格不够现代化

## 3. 设计方案

### 3.1 服务状态控制

**位置**: Header 组件左侧

**组件结构**:
```
[服务状态指示器] [服务名称 + 地址] [启动/停止按钮]
```

**状态指示器**:
- 运行中: 绿色圆点 + "服务运行中" 文字
- 已停止: 灰色圆点 + "服务已停止" 文字

**按钮**:
- 运行中显示 "停止" 按钮（danger variant）
- 已停止显示 "启动" 按钮（primary variant）

### 3.2 组件样式标准化

**Input 组件优化**:
- 边框颜色更柔和（`--line` → 略深色）
- focus 状态添加更明显的边框和阴影
- placeholder 颜色调整
- 错误状态红色边框和阴影

**Button 组件优化**:
- 统一尺寸：small (28px), medium (36px), large (44px)
- 统一内边距：small (8px 12px), medium (10px 16px), large (12px 20px)
- hover 效果：背景色变化 + 边框颜色变化
- active 效果：按下时轻微下沉
- 禁用状态：降低透明度 + 禁用光标

**按钮变体**:
| 变体 | 背景色 | 文字色 | 边框 | 用途 |
|------|--------|--------|------|------|
| primary | accent | white | none | 主要操作 |
| default | transparent | text | line | 次要操作 |
| danger | danger | white | none | 危险操作 |
| ghost | transparent | text | none | 辅助操作 |

### 3.3 导航/Tab 样式优化

**样式调整**:
- 未选中：透明背景 + muted 文字
- 选中：tab-active-bg 背景 + accent 文字 + 底部边框强调
- hover：轻微背景色变化
- 增加图标与文字间距

### 3.4 ServicePage 布局优化

**当前布局**:
```
[侧边栏: 分组列表] [主内容: 规则列表 + 规则详情]
```

**优化点**:
- 侧边栏添加分组搜索/过滤（可选，后续版本）
- 规则列表添加空白状态提示
- 删除按钮改为更显眼的位置（每行操作列）
- 规则表单添加明显的保存按钮

### 3.5 整体视觉风格

**间距系统**:
- 组件内边距：使用 `var(--space-2)` ~ `var(--space-4)`
- 组件间距：使用 `var(--space-3)` ~ `var(--space-6)`
- 页面边距：使用 `var(--space-5)`

**圆角系统**:
- 小型元素：8px
- 中型元素：10px
- 大型元素：12px

**阴影**:
- 卡片：`0 2px 8px rgba(0,0,0,0.08)`
- 弹窗：`var(--shadow)`

- [ ] Button 组件 hover/active 状态有交互反馈
- [ ] 导航 Tab 选中状态清晰可见
- [ ] 规则列表每行有删除操作按钮
- [ ] 规则表单保存按钮样式醒目
